# Social Simulation Open-SAE Replication

This repository packages saved social-simulation experiment outputs and reproducible
Goodfire Open-SAE feature-decomposition scripts.

The goal is to replace the unavailable hosted Goodfire Ember workflow with an
open-source Hugging Face SAE pipeline that others can run on their own GPU.

## What Is Included

- Creativity task saved responses, GPT-5 Torrance judge scores, and Open-SAE features.
- Safe-risk choice saved responses and Open-SAE calibration against the old saved behavior.
- Raw saved ultimatum-game and trust-game outputs for follow-up Open-SAE regeneration.
- Reproducible scripts for GPT judging, Open-SAE inspection, and ablation checks.

No participant responses are regenerated by default. The default scripts read saved CSV
outputs and write derived artifacts.

## Key Caveat: Feature Labels

The original hosted Goodfire Ember natural-language labels are not recoverable from the
open-source SAE weights alone. The old Goodfire API endpoint is currently unavailable.
This repo therefore uses feature indices and activations from:

- Base model: `meta-llama/Llama-3.3-70B-Instruct`
- SAE: `Goodfire/Llama-3.3-70B-Instruct-SAE-l50`
- Hook: `model.layers.50`

Natural-language labels are Neuronpedia/Open-SAE replacement labels. Treat them as
interpretability aids, not exact historical Goodfire Ember label strings.

## Quick Start

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Local checks that do not require a GPU:

```bash
python scripts/run_open_sae_feature_inspection.py \
  --dataset-kind creativity \
  --dry-run \
  --expected-units 320

python scripts/run_open_sae_feature_inspection.py \
  --dataset-kind safe_risky \
  --dry-run \
  --expected-units 4200

python tests/verify_release_artifacts.py
```

GPU rerun commands are in [docs/REPRODUCTION.md](docs/REPRODUCTION.md).

## Public Release Safety

This repo is intentionally a clean export. It should not contain local `.env` files,
API keys, model caches, browser data, or full EDSL source internals.
